# 메뉴 로딩 성능 개선

## 작업 배경
- 메뉴 전환 시 특히 애널리스트 추천 탭에서 로딩 지연이 크게 발생했다.
- 원인은 무거운 `/api/analyst-reports` 요청이 탭 재진입마다 다시 실행되는 구조였다.

## 개선 내용
- `pages/index.tsx`
  - 애널리스트 데이터에 클라이언트 메모리 캐시를 추가했다.
  - 동일한 `days + market` 조합 요청은 5분 동안 재사용한다.
  - 동일 조건의 중복 요청은 inflight promise를 공유해 한 번만 호출한다.
  - 첫 화면 진입 후 idle 시점에 `30일 / 전체시장` 데이터를 백그라운드 프리페치한다.
  - 캐시가 있으면 스피너 없이 즉시 기존 데이터를 먼저 보여주고, 이후 최신 응답으로 갱신한다.

- `pages/api/analyst-reports.ts`
  - 서버 메모리 캐시를 추가했다.
  - 동일한 `days + market` 요청은 5분 동안 재사용한다.
  - 동시에 같은 요청이 들어오면 서버에서도 하나의 inflight 작업을 공유한다.
  - 기존 집계 로직은 `buildAnalystReports()`로 분리해 캐시 경로와 비캐시 경로가 같은 결과를 사용하도록 정리했다.

## 기대 효과
- 애널리스트 탭 재진입 시 재스크래핑 없이 즉시 렌더링된다.
- 필터의 기간/시장 조합을 다시 선택할 때 응답 속도가 크게 줄어든다.
- 동일 시간대의 중복 요청으로 인한 외부 사이트 호출량도 감소한다.

## 검증 포인트
- 첫 진입보다 두 번째 애널리스트 탭 진입 속도가 빨라졌는지 확인
- 같은 기간/시장 조건 재선택 시 스피너 노출이 줄었는지 확인
- API 응답 구조와 필터 결과가 기존과 동일한지 확인
